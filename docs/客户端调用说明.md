# 客户端调用说明

本文总结普通游戏客户端如何接入当前服务端。管理后台走独立的 HTTP 接口，不属于本文范围。

## 1. 基础接入

服务端协议定义在 `src/shared/protocols/serviceProto.ts`，客户端建议直接复用同一份协议。

```ts
import { WsClient } from 'tsrpc';
import { serviceProto } from '../src/shared/protocols/serviceProto';

const client = new WsClient(serviceProto, {
  server: 'ws://127.0.0.1:23414',
  json: true
});

const connected = await client.connect();
if (!connected.isSucc) {
  throw new Error(connected.errMsg);
}
```

说明：

- 普通请求用 `client.callApi(...)`
- 客户端主动推送房间同步消息用 `client.sendMsg(...)`
- 服务端主动下发消息用 `client.listenMsg(...)`

## 2. 推荐初始化顺序

推荐在连接成功后，先注册两个服务端消息监听：

```ts
client.listenMsg('Room/Event', event => {
  console.log('房间事件', event);
});

client.listenMsg('Room/Sync', sync => {
  console.log('房间同步', sync);
});
```

含义：

- `Room/Event`：房间生命周期事件，例如创建房间、玩家加入、准备、倒计时、开局、踢人、房间解散
- `Room/Sync`：房间内的实时业务消息转发，例如状态同步、帧同步、广播消息、定向消息

## 3. 账号相关

### 3.1 注册 `Account/Register`

```ts
const ret = await client.callApi('Account/Register', {
  username: 'alice',
  password: 'password123',
  displayName: 'Alice'
});

if (ret.isSucc) {
  const token = ret.res.session.token;
  const user = ret.res.session.user;
}
```

请求参数：

- `username: string`
- `password: string`
- `displayName?: string`

返回：

- `session.token`：后续鉴权令牌
- `session.user`：当前用户资料

约束：

- 用户名只能是字母、数字、下划线，长度 3 到 24
- 密码长度 6 到 64
- 显示名长度 1 到 24；不传时默认使用用户名

### 3.2 登录 `Account/Login`

```ts
const ret = await client.callApi('Account/Login', {
  username: 'alice',
  password: 'password123'
});

if (ret.isSucc) {
  token = ret.res.session.token;
}
```

返回结构与注册一致。

### 3.3 查询个人资料 `Account/Profile`

```ts
const ret = await client.callApi('Account/Profile', {
  token
});

if (ret.isSucc) {
  console.log(ret.res.user);
}
```

说明：

- 只要接口继承 `AuthenticatedRequest`，就必须传 `token`
- `token` 为空时会报 `Missing auth token`
- 过期或非法会报 `Auth token expired or invalid`

## 4. 房间相关

### 4.1 获取房间列表 `Room/List`

```ts
const ret = await client.callApi('Room/List', {});
if (ret.isSucc) {
  console.log(ret.res.rooms);
}
```

说明：

- 不需要登录
- 返回 `RoomInfo[]`

### 4.2 查询单个房间 `Room/Get`

```ts
const ret = await client.callApi('Room/Get', {
  roomId
});
```

说明：

- 不需要登录
- 用于按房间 ID 拉取最新房间快照

### 4.3 创建房间 `Room/Create`

```ts
const ret = await client.callApi('Room/Create', {
  token,
  name: 'First Room',
  maxPlayers: 4
});

if (ret.isSucc) {
  roomId = ret.res.room.roomId;
}
```

约束：

- `name` 长度 1 到 32
- `maxPlayers` 必须是 2 到 8 的整数
- 同一个用户不能同时在多个房间中

创建成功后，房主会收到一条 `Room/Event`，`type` 为 `room_created`。

### 4.4 加入房间 `Room/Join`

```ts
const ret = await client.callApi('Room/Join', {
  token,
  roomId
});
```

常见失败原因：

- `Room is full`
- `Game already started`
- `User already joined another room`

成功后，房间内成员会收到：

- `player_joined`
- `player_count_changed`

### 4.5 查询自己当前所在房间 `Room/My`

```ts
const ret = await client.callApi('Room/My', { token });
if (ret.isSucc) {
  console.log(ret.res.room);
}
```

说明：

- 未在房间中时返回 `room: null`

### 4.6 离开房间 `Room/Leave`

```ts
const ret = await client.callApi('Room/Leave', {
  token,
  roomId
});
```

说明：

- `roomId` 可选，不传时按当前用户所在房间离开
- 返回：
  - `room`：离开后的房间快照；如果房间被删除则为 `null`
  - `removedRoomId`：如果房间因无人存在而被删除，这里会返回被删除的房间 ID

补充：

- 玩家断线后，如果该用户已经没有在线连接，服务端会自动把他移出房间

### 4.7 准备/取消准备 `Room/SetReady`

```ts
await client.callApi('Room/SetReady', {
  token,
  isReady: true
});
```

说明：

- 只有房间内成员可以调用
- 游戏已经开始时会报 `Game already started`

当所有玩家都准备后，服务端会自动推进房间状态：

- 下发 `countdown_started`
- 连续下发 `countdown_tick`
- 最终下发 `game_started`

如果倒计时期间有人取消准备，会下发 `countdown_canceled`。

## 5. 房间消息同步

### 5.1 客户端主动发送 `Room/ClientSync`

这是低延迟、fire-and-forget 的消息通道，不走 API 返回值承载业务结果。

```ts
const sendRet = await client.sendMsg('Room/ClientSync', {
  token,
  kind: 'state',
  payload: JSON.stringify({ hp: 100 })
});
```

字段说明：

- `token: string`：鉴权令牌
- `kind?: string`：业务消息类别，客户端自定义
- `payload: string`：业务负载，建议统一 `JSON.stringify`
- `targetUserId?: string`：定向发给某个房间成员；不传则广播到整个房间

广播示例：

```ts
await client.sendMsg('Room/ClientSync', {
  token,
  kind: 'chat',
  payload: JSON.stringify({ text: 'hello' })
});
```

定向示例：

```ts
await client.sendMsg('Room/ClientSync', {
  token,
  kind: 'private',
  payload: JSON.stringify({ text: 'only for bob' }),
  targetUserId: bobUserId
});
```

注意：

- `sendMsg` 成功只代表消息已成功写入当前连接发送流程
- 实际业务结果要以服务端下发的 `Room/Sync` 或 `Room/Event` 为准
- 如果 `targetUserId` 不在当前房间，会报 `Target user is not in the room`

### 5.2 客户端接收 `Room/Sync`

```ts
client.listenMsg('Room/Sync', msg => {
  const data = JSON.parse(msg.payload);
  console.log(msg.kind, data);
});
```

服务端返回字段：

- `roomId`
- `fromUserId`
- `fromUsername`
- `fromDisplayName`
- `toUserId`
- `kind`
- `payload`
- `sentAt`

约定建议：

- `payload` 统一使用 JSON 字符串
- 用 `kind` 区分协议版本和业务类型，例如 `state`、`chat`、`frame`、`private`

## 6. 房间事件订阅

客户端应长期监听 `Room/Event`，这是 UI 刷新和状态机推进的主要依据。

```ts
client.listenMsg('Room/Event', event => {
  switch (event.type) {
    case 'player_joined':
    case 'player_left':
    case 'player_kicked':
    case 'player_ready_changed':
    case 'countdown_started':
    case 'countdown_tick':
    case 'countdown_canceled':
    case 'game_started':
    case 'room_dismissed':
      console.log(event.room);
      break;
  }
});
```

当前事件类型：

- `room_created`
- `player_joined`
- `player_left`
- `player_kicked`
- `player_count_changed`
- `player_ready_changed`
- `countdown_started`
- `countdown_tick`
- `countdown_canceled`
- `room_dismissed`
- `game_started`

实战建议：

- 任何房间事件到达后，都以 `event.room` 刷新本地房间状态
- `player_kicked` 可结合 `targetUserId` 判断自己是否被踢出
- `countdown_tick` 可直接驱动倒计时 UI

## 7. 存储相关

### 7.1 写入存储 `Storage/Save`

```ts
const ret = await client.callApi('Storage/Save', {
  token,
  save: {
    hp: '100',
    mp: '40'
  }
});

if (ret.isSucc) {
  console.log(ret.res.savedKeys);
}
```

说明：

- `save` 必须是 JSON 对象
- key 会先 `trim()`
- value 必须是字符串

返回：

- `savedKeys: string[]`

注意：

- 服务端这里是写回缓存模式
- API 返回成功，表示数据已经被接受到内存缓存
- `Storage/Get` 和管理后台会立刻读到最新值
- 如果进程在延迟刷盘前崩溃，最近一次尚未落盘的写入可能丢失

### 7.2 读取存储 `Storage/Get`

```ts
const ret = await client.callApi('Storage/Get', {
  token,
  key: 'hp'
});

if (ret.isSucc) {
  console.log(ret.res.value);
}
```

返回：

- 键存在时返回字符串
- 键不存在时返回 `null`

常见约束：

- key 不能为空
- `Storage/Save` 中的每个 value 都必须是字符串

## 9. 一套最常用的调用流程

```ts
import { WsClient } from 'tsrpc';
import { serviceProto } from '../src/shared/protocols/serviceProto';

async function bootstrap() {
  const client = new WsClient(serviceProto, {
    server: 'ws://127.0.0.1:23414',
    json: true
  });

  client.listenMsg('Room/Event', event => {
    console.log('event', event.type, event.room);
  });

  client.listenMsg('Room/Sync', sync => {
    console.log('sync', sync.kind, JSON.parse(sync.payload));
  });

  const conn = await client.connect();
  if (!conn.isSucc) {
    throw new Error(conn.errMsg);
  }

  const login = await client.callApi('Account/Login', {
    username: 'alice',
    password: 'password123'
  });
  if (!login.isSucc) {
    throw new Error(login.err.message);
  }

  const token = login.res.session.token;

  const createRoom = await client.callApi('Room/Create', {
    token,
    name: 'Demo Room',
    maxPlayers: 4
  });
  if (!createRoom.isSucc) {
    throw new Error(createRoom.err.message);
  }

  await client.callApi('Storage/Save', {
    token,
    save: {
      hp: '100'
    }
  });

  await client.sendMsg('Room/ClientSync', {
    token,
    kind: 'state',
    payload: JSON.stringify({ hp: 100 })
  });
}
```

## 10. 开发建议

- token 建议只在登录或注册成功后更新一次，并统一保存在客户端会话状态中
- 房间 UI 不要只依赖 API 返回值，必须同时消费 `Room/Event`
- `Room/ClientSync` 建议只传输实时业务数据，不要承担关键状态确认
- `payload` 最好统一为 JSON 字符串，避免多端实现分叉
- 房间成员、倒计时、开局状态建议全部以服务端返回的 `RoomInfo` 为准
